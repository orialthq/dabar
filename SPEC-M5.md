# 다바르 M5 — 로컬 AI 전환 기획서

> 목표: API 키 없이, 서버 없이, 모든 사용자의 브라우저에서 동작하는 AI 레이어.

## 0. 배경과 결정

M4는 BYOK(Anthropic API)로 AI를 얹었다. 그러나 타겟(일반 교인)에게 API 키는 구조적 장벽이며, 운영 방향도 API 비의존으로 확정되었다. 이에 AI 레이어를 로컬(브라우저 내) 실행으로 전환한다.

**핵심 아키텍처 결정: 말씀 추천은 생성이 아니라 검색이다.**
- "일상 기록 → 닿는 구절"은 의미 유사도 문제. 챗 LLM 불필요.
- 31,102절을 빌드 타임에 임베딩 → 양자화하여 앱에 동봉 → 브라우저에서 기록을 임베딩 → 코사인 유사도 top-k.
- 효과: 전 사용자 동작(키·서버 불필요), 할루시네이션 원천 불가능(검색이므로), 1회 다운로드 후 오프라인 동작, 비용 0.

**챗(묻다, 묵상 질문 생성)은 선택적 레이어.**
- WebLLM(WebGPU) 기반 소형 모델. 미지원 환경에서는 묻다만 비활성화하고 추천(M5a)은 항상 동작.
- 소형 모델의 성경 참조 암기는 신뢰 불가 → 참조는 임베딩 검색이 공급하고, LLM은 그 위에서 말만 얹는 구조로 역전한다 (아래 M5b).

## 1. 마일스톤 분할

### M5a — 하이브리드 말씀 추천 (코어) — 2026-07-14 구현 완료, 아키텍처 수정됨
> **기획 수정 (2026-07-14, 실측 근거·사용자 승인)**: 순수 임베딩 시맨틱 검색은 일기→약속 구절 매칭에 실패(§5 결정 기록). **주제 앵커 하이브리드**로 전환.

1. **임베딩 파이프라인** (`scripts/embed-bible.mjs`, Node)
   - 모델: `Xenova/multilingual-e5-small` (MIT, 384차원). @huggingface/transformers(Node, fp32)로 실행.
   - 입력 형식: E5 규약 준수 — 절 텍스트는 `passage: {책이름} {장}:{절} {본문}`, 쿼리는 `query: {기록}`.
   - 출력: `public/embeddings/verses-{n}.bin` — int8 양자화(벡터별 scale, float16) + `meta.json`. 실측 총량 11.5MB, 4샤드. 양자화 왕복 오차 min cosine 0.99997(100절 재계산 대조).
2. **주제 앵커 큐레이션** (`scripts/themes.json` + `scripts/embed-themes.mjs`)
   - 30주제 × 일기체 앵커 문장 92개 × 큐레이션 구절 145참조(전수 DB 존재 검증, 교단 중립).
   - 앵커 임베딩 → `public/embeddings/themes.json` (int8+base64, 54KB).
3. **런타임 추천** (`src/lib/semantic.ts`)
   - 브라우저에서 e5-small 로드(WebGPU→WASM 폴백, q8 ~130MB, Cache API 저장, opt-in 진행률 UI).
   - 기록 임베딩 → ① 주제 분류(앵커 코사인 최대값) → 1위 주제 큐레이션 2건 + 2위 주제 1건 ② 31k 전 절 시맨틱 top-8 중 장 미중복 1건을 "발견" 슬롯으로 = 총 4건.
   - why 문구 = 주제 라벨("불안과 염려" 등), 발견 슬롯은 "발견" 태그.
4. **UI**: `src/components/VerseSuggest.tsx` 신규(M4 코드는 리포에 없어 교체가 아닌 신규 작성). "이 하루에 닿는 말씀 찾기" UX.
5. **폴백**: 모델 로드 실패/거부/미지원 시 기존 키워드 검색(VersePicker)으로 안내. 기능 전체가 죽지 않음.

### M5b — WebLLM 챗 레이어 (선택적) — 2026-07-14 구현 완료
1. **엔진 추상화** (`src/lib/engine.ts`): `generate(system, messages) → text` + 스트리밍/진행률/temperature 옵션. 구현체 3종:
   - `webllm`: @mlc-ai/web-llm 0.2.84, 기본 **Qwen3-4B-q4f16_1-MLC** (프리빌드 확인, ~2.3GB·VRAM 3.4GB). 설정에서 Qwen3.5-4B / Qwen3-1.7B 선택 가능. thinking은 enable_thinking=false + `<think>` 스트립 이중 차단. Kanana nano는 MLC 프리빌드 부재로 보류.
   - `ollama`: 주소·모델명 설정 가능 (개발·파워유저용)
   - `anthropic`: BYOK 신규 작성 (M4 코드 리포 부재 — 옵션으로 존치, 외부 전송 명시)
   - `src/pages/Settings.tsx`(#/settings)에서 선택. 기본값 webllm.
2. **역전된 RAG** (`src/lib/ask.ts` + `semantic.ts retrieveForAsk`): ① 질문 임베딩 → 1위 주제 큐레이션 3 + 시맨틱 5절을 **본문 포함** 프롬프트 공급 ② LLM은 그 안에서만 근거 ③ 답변 내 인용 금지, "(책 장:절)" 참조만 → `resolveRefs`가 **공급 목록 화이트리스트 + DB 실존** 이중 검증 후 로컬 DB 렌더링.
3. **묻다** (`src/pages/Ask.tsx`, #/ask): 스트리밍 챗, 최초 다운로드 동의·진행률, WebGPU 미지원 시 안내 + 대체 엔진 유도. **묵상 질문** (`src/lib/meditate.ts` + Editor의 MeditationPrompt): 첨부 구절+기록 → 열린 질문 2~3개, "담기"로 본문에 덧붙임.
4. 관찰: Qwen3-4B 한국어가 간혹 어색(오타·영어 혼입) — temperature 0.5(묵상)로 보정, 필요 시 Qwen3.5-4B 승격 검토.

## 2. 불변 조건 (변경 없음)
- 구절 본문은 항상 로컬 성경 DB에서만 렌더링. LLM 생성/복사 본문 표시 금지.
- 개역한글 외 역본 금지. 본문 무변형.
- 서버 금지. 저장은 localStorage. 외부 전송은 사용자가 명시 선택한 엔진 호출뿐.
- 교단 중립, 정답 제시가 아닌 상고 유도 (기존 프롬프트 원칙 유지).

## 3. 검증 기준 — M5a 결과 (2026-07-14)
- 파이프라인: 임베딩 절 수 = 31,102 ✓, meta 매핑 전수 일치 ✓, 무작위 100절 재계산 대조 평균 cosine 0.999983 / 최소 0.999969 ✓ (`scripts/verify-embeddings.mjs`).
- 추천 품질 골든셋(`scripts/goldenset.json`, 10건): 순수 dense 0/10 → **하이브리드 9/10** (`scripts/eval-goldenset-hybrid.mjs`). 유일 실패(간병)도 주제 분류는 정확, 반환 구절 목회적 적합.
- 성능: 검색 1회 751ms(M2 Mac WebGPU, 최초 호출 포함) < 1.5s ✓. WASM 중급 노트북·iOS 실기기 측정은 잔여 항목.
- 회귀: 리더/검색/새김/첨부 브라우저 확인 ✓ (내보내기는 코드 경로 무변경).

## 5. 결정 기록 (2026-07-14) — 순수 dense → 하이브리드 전환
골든셋 10건 기준 실측 (기대 구절 최고 랭크 평균 / 31,102절 중, `scripts/exp-model-screen.mjs` 표본 3,000 추정·실측 검증 오차 ±10%):

| 방식 | 다운로드 | 최고랭크 평균 | top-4 적중 |
|---|---|---|---|
| e5-small 절 단독 | 118MB | 1,515위 | 0/10 (실측) |
| e5-small 문맥 포함(±1절) | 118MB | 유사 | 0/10 (실측) |

| e5-base | 281MB | 561위 | 0/10 |
| e5-large (q8) | ~560MB | 585위 | 1/10 |

원인: 일기 텍스트는 임베딩 공간에서 "상황을 묘사하는 절"과 가깝고 "응답하는 약속의 절"과는 멀다. 모델 확대(5배)로도 개선 미미 → 구조 문제. 반면 일기→일기체 앵커(same-domain) 매칭은 e5-small로 충분. 전 절 인덱스는 발견 슬롯과 M5b 역전 RAG에 계속 사용.

## 4. 리스크
- 모델 최초 다운로드(~100MB) 이탈 → 다운로드 전에도 앱 가치가 성립하도록(리더+키워드검색+수동 새김), 추천 기능은 "준비하기" 버튼으로 opt-in.
- iOS Safari WebGPU/메모리 제약 → WASM 경로 필수 테스트.
- Qwen3 소형 모델의 한국어 존댓말 톤 → 시스템 프롬프트로 보정, 골든셋으로 확인.

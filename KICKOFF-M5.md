다바르(dabar) 프로젝트의 M5를 진행한다. 시작 전에 반드시 다음을 순서대로 읽어라:

1. `CLAUDE.md` — 절대 규칙과 컨벤션. 특히 "구절 본문은 로컬 DB에서만 렌더링" 불변 조건.
2. `SPEC-M5.md` — 이번 마일스톤 전체 기획. M5a(임베딩 추천)와 M5b(WebLLM 챗)로 분할되어 있다.
3. `scripts/DATA.md` — 성경 데이터 구조와 무결성 조건.

작업 지시:
- **M5a부터 시작한다. M5a 완료·확인 전에 M5b에 착수하지 마라.**
- 첫 단계는 `scripts/embed-bible.mjs` 임베딩 파이프라인이다. multilingual-e5-small의 MLC/ONNX 가용성, E5 프리픽스 규약, int8 양자화 포맷을 먼저 검증하는 최소 스크립트를 만들고, 소량(창세기 1권)으로 end-to-end를 확인한 뒤 전체 66권으로 확장하라.
- SPEC-M5 §3의 검증 기준(절 수 31,102 일치, 무작위 재계산 대조, 골든셋 10건)을 통과해야 M5a 종료다. 골든셋 시나리오는 네가 초안을 만들고 나에게 확인받아라.
- 기존 M1~M4 기능이 깨지면 안 된다. 각 커밋 전 `npm run build` 통과 필수.
- 브라우저 모델 다운로드 UX(진행률, opt-in, 실패 폴백)는 SPEC-M5 §1-2와 §4를 따르라.
- 결정이 필요한 갈림길(모델 교체, 용량 트레이드오프 등)에서는 임의로 정하지 말고 근거와 함께 나에게 물어라.

먼저 SPEC-M5를 읽고, M5a의 작업 계획(파일 단위)과 예상 리스크를 요약해서 보고한 뒤 시작하라.

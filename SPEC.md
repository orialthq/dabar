# 다바르 (dabar) — 기획서

> 오늘 있었던 일을 쓰면, 성경이 그 하루에 답하는 신앙 기록 서비스

- 레포: `orialthq/dabar` · 배포: GitHub Pages (`orialthq.github.io/dabar`)
- 상태: M1 (스캐폴드 + 배포 파이프라인)

## 1. 네이밍

**다바르(דָּבָר)** 는 히브리어로 '말씀'과 '사건(일어난 일)'을 동시에 뜻한다. 서비스의 핵심이 "일상의 사건 ↔ 성경 말씀의 연결"이므로, 그 이중성이 단어 하나에 내장되어 있다. 한 줄 슬로건: **"당신의 다바르를, 다바르로."**

- 검증: 마이블(Mible, 온맘닷컴 2009)과 동명 충돌로 마이블 탈락. 베레아(이단 연상), 레마(은사주의 색채), 셀라(고유성 부족) 탈락.
- 로마자 `dabar`, 발음 모호성 없음.

## 2. 문제 정의와 타겟

- 타겟: QT/묵상 습관이 있거나 갖고 싶은 한국 개신교인. 상황 프레임 — "오늘 하루가 마음에 걸리는데, 말씀으로 정리하고 싶은 사람".
- 문제: 기존 성경 앱은 리더+통독 관리 중심. "내 일상 → 말씀 → 나의 통찰"을 잇는 기록 도구는 없다.
- 행동 기반: QT는 이미 확립된 습관. 새 행동을 만들지 않고 기존 습관의 도구를 바꾼다.

## 3. 핵심 기능 (세 가지 동사)

| 동사 | 기능 | 설명 | 마일스톤 |
|---|---|---|---|
| 읽다 | 성경 리더 + 검색 | 개역한글 전문, 책/장/절 탐색, 키워드 검색. 구절은 항상 DB 원문 그대로 표시 | M2 |
| 새기다 | 다바르 기록 (킬러) | 일상 기록 → 관련 구절 연결 → 묵상 질문 → 통찰 기록. 타임라인으로 축적 | M3 |
| 묻다 | 말씀에 묻기 | 키워드 질문, 해석 검증. 답변은 반드시 실제 구절 인용과 함께 | M4 (AI) |

AI 없는 상태에서도 [읽다]와 수동 [새기다]가 완결적으로 동작해야 한다. AI는 레이어다.

## 4. 저작권 정책 (핵심 제약)

- 본문: **성경전서 개역한글판(1961)** — 저작재산권 보호기간 만료, 사용료 없음.
- 준수 사항: **동일성유지권**(본문 무변형 — 한 글자도 바꾸지 않음), **성명표시권**(대한성서공회 역 표기).
- LLM이 구절을 기억으로 생성하는 것 금지. 반드시 로컬 성경 DB에서 원문을 조회(RAG)해 그대로 표시한다. (저작권 + 할루시네이션 동시 차단)
- 개역개정 등 타 역본은 대한성서공회 라이선스 협의 후에만 도입 가능. MVP 범위 외.

## 5. 아키텍처

- 프론트: Vite + React 18 + TypeScript strict + Tailwind CSS 4
- 호스팅: GitHub Pages (GitHub Actions 자동 배포)
- 성경 데이터: 책 단위 정적 JSON (`/public/bible/{book}.json`), 필요 시 fetch + 캐시
- 기록: localStorage + 마크다운 export (계정 없음)
- AI: Phase 1 — BYOK(사용자 API 키, localStorage 저장). Phase 2 — Cloudflare Workers 무료 티어 프록시로 키 문제 해소
- 비용: 0원 (OriAlt 원칙)

주의: BYOK는 일반 교인 타겟과 구조적으로 안 맞는다. Phase 1은 본인용+검증용으로 규정하고, 신호 확인 후 Phase 2로 해소한다.

## 6. 마일스톤

- **M1**: 스캐폴드, 디자인 토큰, 랜딩 셸, GitHub Pages 배포 파이프라인 검증 ← 현재
- **M2**: 개역한글 데이터 파이프라인(원문 → JSON 변환 스크립트), 리더 UI, 키워드 검색
- **M3**: 새김 기록(작성/타임라인/구절 첨부/export), localStorage 스키마
- **M4**: AI 연결(BYOK) — 일상 기록 → 구절 추천 + 묵상 질문, 해석 검증 대화
- **M5+**: Cloudflare Workers 프록시, 목회자 감수 가이드라인, 커스텀 도메인

## 7. 디자인

- 컨셉: **새벽** — 새벽기도의 시간. 잉크(새벽 하늘)와 한지(기록), 동트는 빛(액센트)
- 토큰: ink `#10151F` / ink-soft `#1B2230` / hanji `#F7F3EA` / hanji-dim `#EFE9DB` / dawn `#D98E32` / mist `#8A94A6`
- 타입: Noto Serif KR(말씀·디스플레이) + Pretendard(본문)
- 시그니처: 히브리어 דָּבָר 타이포와 "말씀/사건" 이중 의미의 병치

## 8. 리스크

1. **신학적 신뢰**: AI가 해석 주체로 보이면 교계 반발. 포지셔닝은 "정답을 주는 AI"가 아니라 "구절을 연결하고 스스로 묵상하게 돕는 도구". 교리 논쟁 지점(구원론, 은사 등)에서는 중립 + 구절 제시로 한정. 장기적으로 목회자 감수 필요.
2. **인용 정확성**: RAG 강제만으로 부족하면 표시 전 원문 대조 검증 단계 추가.
3. **BYOK 진입장벽**: Phase 2에서 해소. Phase 1 지표는 AI 사용률이 아니라 기록 리텐션.
